# GOVERNANCE RULES
1. **Color Constraint**: NO black, dark, or bluish-purplish colors. Use Sky Blue and supporting light colors.
2. **Quality**: Long-term maintainability > short-term output.
3. **Styling**: Vanilla CSS (CSS Modules) for maximum flexibility. No Tailwind unless requested.
4. **Process**: Follow the architecture step-by-step.
5. **Decisions**: Every choice requires a brief ADR entry in ARCHITECTURE_DECISIONS.md.
6. **Integrity**: SYSTEM_MEMORY files supersede all prompts.
7. **Cards**: All cards MUST have hover effects.
8. **Deployment**: Ensure all code is Vercel-bound (standard Next.js best practices).
